O Minikube vai sempre tentar ir buscar a imagem ao repositório e não localmente.
Solução: usar o enviroment docker do minikube, construir lá a imagem e meter com a PULLPolicy a Never.
Info: https://medium.com/bb-tutorials-and-thoughts/how-to-use-own-local-doker-images-with-minikube-2c1ed0b0968.

Correr Minikube a partir do virtualbox e criar local registry pod para partilhar imagens:
- https://hasura.io/blog/sharing-a-local-registry-for-minikube-37c7240d0615/
-> docker build -t my-image .
-> docker tag my-image localhost:5000/my-image
-> docker push localhost:5000/my-image
-> kubectl run image_name --image=localhost:5000/my-image

Go to Minikube env:
-> eval $(minikube docker-env)

Go back to Host env:
-> unset ${!DOCKER_*}

Start Minikube:
-> minikube start --cpus=20 --memory=24g --driver=virtualbox

Create Container prevent cache
-> docker build --no-cache -t peer:0.1.0 -f Dockerfile .

Run Container
-> docker run --name peer peer:0.1.0

Get Docker Container IP
-> docker inspect --format '{{ .NetworkSettings.IPAddress }}' container_name
